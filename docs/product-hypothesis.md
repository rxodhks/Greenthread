# Greenthread 제품 가설 (MVP 이후 기준)

내부 정렬용 문서입니다. 우선순위가 바뀌면 이 파일을 업데이트합니다.

## 다음 분기 축 (실행 확정)

**복합 축 (B2C 유지)**: **운영(집계)** + **성장(공유)** 를 같이 밀고, **신뢰(개인정보·AI 고지 페이지)** 를 병행한다.  
**B2B 분기**는 1순위 사용자 가설이 바뀔 때 별도 스프린트로 잡는다(감사 로그·데모 예약 등).

구현 메모: `/admin/metrics` + `GET /api/admin/metrics`(Bearer `METRICS_ADMIN_SECRET`), 공유 카드 `/share/[id]`, `/privacy`, 주간 JSON 출력 `npm run telemetry:week`(환경 변수 `BASE_URL`, `METRICS_ADMIN_SECRET`).

## 다음 스프린트 단일 목표 (선택 확정)

**1순위: 학습(측정·CTA)** — 스캔 이후 행동을 숫자로 보고, CTA를 실제 링크로 연결한다.  
**2순위: 지속(저장소)** — 스레드·텔레메트리를 Upstash Redis 등 관리형 저장소에 두어 배포/스케일 후에도 맵이 일관되게 동작한다.  
**3순위: 신뢰(출처)** — 시나리오별 계수 출처·버전을 UI에 노출한다.  
**4순위: 성능·모바일** — 지도 TopoJSON을 지연 로드하고, 업로드 이미지를 모바일 친화적으로 줄인다.

범위 밖(당분기): WebSocket 실시간 타일 지도, 실제 결제 정산, 법적 기부 매칭 계약.

## 이번 분기 성공 기준 (가설 수치)

- **측정**: 스캔 성공 세션 대비 **CTA(기부 또는 대체 행동) 클릭률 ≥ 15%**를 1차 목표로 두고, `POST /api/telemetry` 이벤트로 집계한다. (실데이터 전까지는 가설)

## 기술 부채·릴리스 메모

- Upstash Redis(`UPSTASH_REDIS_*`) 미설정 시 스레드·텔레메트리는 프로세스 메모리/개발 콘솔에 한정된다.
- 레이트 리밋은 인스턴스 로컬이므로 대규모 남용 방지에는 엣지/게이트웨이가 필요하다.

## 1순위 사용자 (현재 가설)

- **1차**: 탄소·공급망 정보에 관심 있는 **일반 소비자**(모바일 스캔·교육형 인사이트).
- **2차(확장 시)**: ESG·구매 담당 **B2B**(리포트·감사 가능한 출처).
- **파트너**: 검증 가능한 **NGO/프로젝트**와 1:1 연계 카피는 실제 계약 전까지 **예시·교육용**으로 유지.

## 전환 행동 (측정 후보)

- 스캔 완료 후 **소액 기부** 또는 **챌린지/대체 행동** CTA 클릭.
- **결과 공유**(스크린샷·링크) — 구현 시 이벤트 로깅 추가.
- B2B 전환 시 **데모 문의** 등 별도 퍼널.

## 수치·NGO 문구 수준 (법·커뮤니케이션)

- CO₂e 및 NGO 임팩트는 **모델·계수 기반 추정**이며, **법적 보증이나 공식 인증을 대체하지 않음**.
- UI에는 **“추정”**, **출처/면책**, **제3자 AI 처리 시 사전 고지**를 노출한다.
- 실제 기부·매칭은 별도 약관·검증 프로세스가 필요하며, MVP 이후 단계에서 정의한다.

## 엔지니어링 함의

- 소비자 우선: 모바일 스캔 UX, 업로드 제한, 프라이버시 고지.
- B2B 우선이 되면: 출처 메타데이터, 감사 로그, 대시보드가 같은 추정 파이프라인보다 앞설 수 있음.
